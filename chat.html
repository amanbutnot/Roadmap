📋 Project Roadmap

Secure Messenger

Because building WhatsApp wasn't ambitious enough.

This roadmap breaks the entire project into 8 versioned phases, each building on the last. You start with a working messenger (v1.0), layer in quality-of-life features, introduce AI, harden security, go offline, support multiple devices, add real-time calls, and finally land full desktop parity in v4.0. Every feature has a checkbox — tick it when done. The overall progress bar updates automatically.

Click "Show features" on any card to expand it. Each item includes a plain-English explanation of what it does and why it matters technically. Work top-to-bottom — later phases depend on earlier ones.

Execution Plan
v1.0
Core System
Foundation — where everything either starts or collapses
HIGH

The essential plumbing: user accounts, real-time messaging, group chats, media, push notifications, and offline-first local storage. Without this, nothing else exists.

v1.1
Usability
Make it usable — because functional and enjoyable are different things
MEDIUM

↳ requires v1.0

The features users actually notice: editing messages, reacting with emojis, voice messages, link previews, a media viewer, and privacy controls. This is what separates a working app from one people want to use.

v1.5
AI Layer
Python enters the chat — and immediately asks for a requirements.txt
HIGH

↳ requires v1.1

Smart replies, chat summarization, translation, voice transcription, and optional sentiment detection. AI as a useful tool, not a gimmick — and always opt-in.

v1.6
Power Features
Control your data — because ownership matters
MEDIUM

↳ requires v1.1

Full-text search, media filters, chat folders, starred messages, and a storage manager. The features that transform "it works" into "it works exactly how I want it to."

v2.0
Security
Now things break — and then we fix them properly
VERY HIGH

↳ requires v1.0 — retrofit encryption into existing message flow

End-to-end encryption via the Signal Protocol, encrypted media, proper key management, QR code identity verification, screenshot detection, and self-destructing messages. The hardest phase to build and the most important one to get right.

v2.5
Multi-Device
Welcome to pain — consistency across chaos
VERY HIGH

↳ requires v2.0 — per-device E2E keys depend on encryption being in place

Link up to 5 devices, sync messages and read status across all of them in real time, maintain per-device encryption so a compromised tablet doesn't compromise everything, and manage sessions from one screen.

v3.0
Offline System
Distributed systems hell — where CAP theorem laughs at you
VERY HIGH

↳ requires v2.0 — offline messages still need to be encrypted

Message without the internet: LAN messaging via mDNS, Bluetooth messaging over BLE, a persistent offline queue, cross-transport sync when connectivity returns, and deduplication so a message delivered via three paths only appears once.

v3.5
Calls
Real-time communication — milliseconds matter now
HIGH

↳ requires v2.0 — call signalling must be encrypted; v2.5 for multi-device call routing

Encrypted voice and video calls via WebRTC, group calls via an SFU that routes without decrypting, and adaptive bitrate that keeps the call alive when the network is being difficult.

v4.0
Full Platform
Final boss — where desktop gets the respect it deserves
HIGH

↳ requires v3.5 — screen sharing & LAN calls build on the call infrastructure

Screen sharing, call recording, LAN/Bluetooth calls, full Windows/macOS/Linux desktop parity, drag-and-drop, and a system tray presence. The app stops being a phone app and becomes a platform.
