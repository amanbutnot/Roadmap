🍎 SwiftUI Mastery Roadmap

The complete path from "what is SwiftUI?" to building production-grade iOS/macOS apps — with every concept you need to ship to the App Store.
SwiftUI was unveiled by Apple in 2019 and is now the preferred framework for building apps across all Apple platforms: iOS, macOS, watchOS, tvOS, and visionOS. It uses a declarative syntax — you describe what the UI should look like, and SwiftUI handles the rest. This roadmap assumes zero experience.

PHASE 0

⚙️ Phase 0 — Setup & Swift Basics

Goal: Get Xcode installed, understand the Swift playground environment, and write your first working SwiftUI view.

Module 0.1 — Installing Xcode & Setup

Download Xcode from the Mac App Store — confirm you have Xcode 15+ for the latest SwiftUI features
Install iOS Simulator — run a default blank app to confirm your setup works
Understand the Xcode layout: Navigator, Editor, Inspectors, Debug area, Canvas (Previews)
Enable Developer Mode on a real iPhone/iPad if you have one
Create a new iOS project with the SwiftUI App template — explore what gets generated

Module 0.2 — Swift Language Basics

Variables: var (mutable) vs let (constant) — prefer let by default
Basic types: Int, Double, Float, String, Bool, Character
Type inference — Swift infers type from assignment: let name = "Alice"
String interpolation: "Hello, \(name)!"
Optionals: var age: Int? = nil — understand why Swift requires explicit optionals
Unwrapping: if let, guard let, ?? (nil coalescing), force unwrap ! (use rarely)
Control flow: if/else, switch, for-in, while
Functions: func greet(name: String) -> String { ... }
Closures: { (x: Int) -> Int in return x * 2 } and trailing closure syntax
let name = "SwiftUI"
let message = "Hello, \(name)!"
var score: Int? = 42
let finalScore = score ?? 0  // nil coalescing

Module 0.3 — Collections & Structs

Arrays: var items: [String] = [] — append, remove, iterate with for-in
Dictionaries: var dict: [String: Int] = [:] — access, update, iterate
Sets: understand uniqueness — when to use Set vs Array
Structs: struct User { var name: String; var age: Int } — value semantics
Classes vs Structs — reference types vs value types, why SwiftUI models prefer structs
Enums: enum Direction { case north, south, east, west }
Enums with associated values: case success(String), failure(Error)
Computed properties: var fullName: String { "\(first) \(last)" }

Module 0.4 — Protocols & Extensions

Protocols define a blueprint: protocol Describable { var description: String { get } }
Conform a struct to a protocol: struct Product: Describable { ... }
Key SwiftUI protocols: Identifiable, Hashable, Codable, Equatable
Extensions: add methods/computed properties to existing types without subclassing
Protocol extensions — provide default implementations
Generics: func first<T>(of items: [T]) -> T? { items.first }

Module 0.5 — Your First SwiftUI View

Understand that every SwiftUI view is a struct conforming to View protocol
The body property returns some View — understand opaque return types
Use Xcode Canvas (Preview) — #Preview { ContentView() }
Display text: Text("Hello, SwiftUI!")
Apply modifiers: .font(.title), .foregroundColor(.blue), .padding()
Run on iPhone Simulator — confirm your view renders correctly
struct ContentView: View {
    var body: some View {
        Text("Hello, SwiftUI!")
            .font(.largeTitle)
            .foregroundColor(.orange)
            .padding()
    }
}

PHASE 3

🔄 Phase 3 — State & Data Flow

Goal: Master SwiftUI's reactivity system — the property wrappers that make your UI update automatically when data changes.

Module 3.1 — @State & @Binding

@State is source of truth for simple local view state — always private
SwiftUI re-renders the view body when @State changes — understand the diffing process
@Binding creates a two-way connection to a parent's state: @Binding var isOn: Bool
Pass binding with $ prefix: ChildView(isOn: $isOn)
Binding.constant(true) — create a read-only binding for previews
Computed bindings: derive a Binding from a stored value using Binding(get:set:)
struct CounterView: View {
    @State private var count = 0
    var body: some View {
        Button("Count: \(count)") { count += 1 }
    }
}

Module 3.2 — ObservableObject & @Published

Create a class that conforms to ObservableObject — your view model
Mark properties with @Published — any change triggers UI updates
@StateObject — own and create an observable object: @StateObject private var vm = ViewModel()
@ObservedObject — receive (not own) an observable object passed from parent
Understand ownership: use @StateObject for creation, @ObservedObject for injection
Manual publishing: objectWillChange.send() for non-@Published changes

Module 3.3 — @Observable (Swift 5.9 / iOS 17+)

The new @Observable macro replaces ObservableObject — simpler syntax, better performance
No need for @Published — all stored properties are automatically tracked
Use @State with @Observable classes: @State private var model = MyModel()
@Bindable — create bindings from @Observable objects: @Bindable var model
Know when to target iOS 16 (use ObservableObject) vs iOS 17+ (use @Observable)

Module 3.4 — @EnvironmentObject & Environment

@EnvironmentObject injects a shared object deep into the view hierarchy — no prop drilling
Inject at root: .environmentObject(myStore) on a parent view
Read anywhere: @EnvironmentObject var store: AppStore — crash if not injected (common mistake)
Custom environment values: define a key with EnvironmentKey, inject with .environment(\.myKey, value)
iOS 17+: use .environment(myObservableObject) directly

Module 3.5 — Combine Basics for SwiftUI

Combine is Apple's reactive framework — Publishers emit values, Subscribers react
Common publishers: Just, Future, PassthroughSubject, CurrentValueSubject
Operators: .map, .filter, .debounce, .removeDuplicates, .sink
Store subscriptions: var cancellables = Set<AnyCancellable>()
Debounce search input: $searchText.debounce(for: .milliseconds(300), scheduler: DispatchQueue.main)

Module 3.6 — async/await & Task in Views

Swift Concurrency: async functions, await keyword, structured concurrency
.task { } modifier — runs async work when view appears, cancels on disappear
Task { } in button actions — fire unstructured async work
@MainActor — ensure UI updates happen on main thread
withAnimation { } inside async context — animate state changes from async code

func fetchUser(id: Int) async throws -> User {
    let url = URL(string: "https://api.example.com/users/\(id)")!
    let (data, _) = try await URLSession.shared.data(from: url)
    return try JSONDecoder().decode(User.self, from: data)
}
